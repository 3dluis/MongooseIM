1. Introduction

List of all user's conversations along with number of unread messages is very desirable feature especially in mobile clients. This experience can partially be achieved with Message Archive Management (XEP-0313) [1] and roster but it is not very convenient. For every user from the contact list, the client has query the archive or read the archive without filtering by JID wich may not be handy.

This extensions provides simple protocol to read all user's conversations with number of unread messages for each item from the inbox. Optionally the server may return last message for each conversation if the clinets asks for it.

2. Requirements

Both servers and clients has to understand Message Archive Management (XEP-0313) [2] as this extensions assumes that following messages sync is done with MAM. Also the server has to add the ID of a message as used in the archive as described in Publishing Available Jingle Sessions (XEP-0358) [3], this is to allow users to mark a message as read.

3. Use Cases

3.1 Reading the inbox

Reading the inbox is as simple as sending following IQ to the server:

Example 1. Quering the inbox

<iq type='get' id='inbox1'>
  <query xmlns='urn:xmpp:inbox:1'/>
</iq>

Example 2. Server returns conversations list

<iq type='result' from='alice@wonderland.lit' to='alice@wonderland.lit/resource'>
  <query xmlns='urn:xmpp:inbox:1'>
    <item with='rabbit@wonderland.lit' unread='3' stamp='2016-12-31T20:00:27Z'/>
    <item with='rome@shekspear.lit' unread='0' stamp='2017-01-02T12:34:32Z'/>
  </query>
</iq>

The stamp attribute in every item contains the timestamp (as seen by ther server) of the last read message. This value can be later used by the user to query MAM to read all unread messages with specific contact.

3.2 Reading the inbox with last message

The server MAY provide an extended inbox view where for each conversation the content of last message is included.

This optional query may proof usefull if the application UI displays last message for each conversation. With this query the result will be bigger, but the client will need to send only one request to the server.

Example 3. Quering the inbox with last message for each conversation

<iq type='get' id='inbox2'>
  <query xmlns='urn:xmpp:inbox:1'>
    <x xmlns='jabber:x:data' type='submit'>
      <field var='FORM_TYPE' type='hidden'>
        <value>urn:xmpp:inbox:1</value>
      </field>
      <field var='last-message'>
        <value>1</value>
      </field>
    </x>
  </query>
</iq>

Example 4. Server returns conversations list with last messages

<iq type='result' from='alice@wonderland.lit' to='alice@wonderland.lit/resource'>
  <query xmlns='urn:xmpp:inbox:1'>
    <item with='rabbit@wonderland.lit' unread='3' stamp='2016-12-31T20:00:27Z'>
      <message from='alice@wonderland.lit' to='rabbit@wonderlang.lit' type='chat'>
        <body>Hi rabbit, what are your New Year's Eve plans?</body>
      </message>
    </item>
    <item with='rome@shekspear.lit' unread='0' stamp='2017-01-02T12:34:32Z'>
      <message from='rome@shekspear.lit' to='alice@wonderlang.lit' type='chat'>
        <body>Happy New Year!</body>
      </message>
    </item>
  </query>
</iq>
